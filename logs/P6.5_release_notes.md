# P6.5 First Public Beta Release Notes

Updated: 2026-04-01
Stage: P6.5 Public Beta Release
Status: P6.5-7 Complete
Next Stage: P7 Codex Support

## What This Beta Is

This is the first public Beta for the current control-plane baseline. It is a
small, technical, source-run / source-install Beta for limited external trial,
not a finished packaged product.

The current Beta is designed to let a small group of technical evaluators run
one real path:

- start local `relay`
- start local `desktop`
- install and run remote Linux `remote-agent`
- launch `Kimi --wire` through `remote-agent`
- observe one hosted session and one approval path in desktop
- complete one `Approve` or `Reject`

## Included In This Beta

- local Windows `desktop`
- local `relay`
- remote Linux `remote-agent`
- `Kimi --wire` as the only supported provider
- hosted session CLI baseline
- multi-remote baseline
- operator-led small-group trial path
- issue templates for bug, trial, and environment feedback

## Not Included In This Beta

- `Codex`
- `Claude Code`
- installer, packaged desktop app, or signed distribution
- auto-update
- `attach`
- persisted recovery, replay, or `P8` reliability work
- remote deploy targets beyond Linux
- local packaged experience beyond source-run

## Supported Platforms And Provider

- local `desktop + relay`: Windows
- remote `remote-agent`: Linux
- provider: `Kimi --wire` only

This Beta must not be described as shipping local macOS, local Linux,
Windows/macOS remote-agent deploy, or multi-provider support.

## Known Limitations

- `desktop` is source-run only
- local `relay` is started manually
- remote `remote-agent` still needs manual env editing
- network reachability still needs manual confirmation from both sides
- `install-systemd-user.sh` does not fill
  `REMOTE_AGENT_RELAY_ENDPOINT`,
  `REMOTE_AGENT_CONTROL_BASE_URL`, or `REMOTE_AGENT_REMOTE_NAME`
- `watch` is single-read, not continuous follow
- `attach` is not implemented
- `stop` rejects `approval_pending` and running-turn sessions
- `relay` and `remote-agent` are still in-memory; restart does not restore
  prior control-plane state
- no provider execution-state `resume / reattach` promise exists

## External Core Wording

The truthful outward-facing summary for this Beta is:

This first public Beta is a small technical trial release for the current
`Windows local desktop + relay` and `Linux remote-agent` path, with
`Kimi --wire` as the only supported provider. It is intended for source-run /
source-install evaluation of the hosted-session and approval flow, not for
installer-based general release.
