## 当前 `v1.0` 是什么

当前 `v1.0` 是一条自托管、source-run / source-install 的正式发布面，覆盖：

- 本地 Windows `desktop`
- 本地 `relay`
- 远端 Linux `remote-agent`
- `Kimi --wire`
- multi-remote 聚合
- 本地 session detail / reply / approval 控制闭环

它不是 installer 版桌面产品，也不是云服务版控制平面。

## 当前 `v1.0` 包含

- 本地 Windows source-run `desktop`
- 本地 operator-run `relay`
- 远端 Linux `remote-agent`
- `Kimi --wire` hosted session 启动
- 本地 session 列表
- 本地 session detail / recent transcript
- 本地 UI reply
- 本地 UI `approve / reject`
- multi-remote 聚合
- `remote_id + request_id` 唯一定位 approval
- 远端 `remote-agent sessions / watch / reply / stop`

## 当前 `v1.0` 不包含

- `Codex`
- `Claude`
- `attach`
- reconnect / checkpoint / replay
- pending approvals replay
- `remote-agent` 重启恢复
- provider 执行现场恢复
- installer / 签名分发 / 自动更新
- 云 relay / 云服务 / 多设备 / 账号体系
- 通用聊天工作台 / token 流透传 / 推理链可视化

## 当前正式支持面

- 本地：Windows
- 远端：Linux
- provider：`Kimi --wire`

当前不能写成已支持：

- 本地 macOS 正式交付
- 本地 Linux 正式交付
- 远端 Windows / macOS deploy
- 多 provider 正式支持

## 最小使用路径

1. 本地启动 `relay`
2. 本地启动 `desktop`
3. 远端安装并启动 `remote-agent`
4. 远端启动一个 `Kimi` hosted session
5. 在本地 UI 中看到 session 并打开 detail
6. 在本地 UI 中提交一轮 `reply`
7. 在本地 UI 中处理一轮 approval

推荐用于验证当前 `reply -> approval` 闭环的 UI reply：

```text
Use the shell tool to run pwd and return only the absolute path. Do not answer from memory.
```

## 已知限制

- `desktop` 仍是 source-run
- `relay` 仍需手工启动
- `remote-agent` 仍需手工补 env
- 网络连通性仍需手工确认
- `desktop` 当前以手工 refresh 为主
- `watch` 是单次读取，不是持续 follow
- `attach` 未实现
- `stop` 不能在 `approval_pending` 或 turn 运行中执行
- `relay` 与 `remote-agent` 都仍是内存态
- 当前不承诺 provider `resume / reattach`
