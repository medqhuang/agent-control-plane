# P6.5 First Public Beta Launch Checklist

Updated: 2026-04-01
Stage: P6.5 Public Beta Release
Status: P6.5-7 Complete
Next Stage: P7 Codex Support

Use this checklist before describing the first public Beta as ready for limited
external trial.

## 1. Scope And Messaging

- [ ] release messaging says this is a first public Beta, not a finished product
- [ ] release messaging says the Beta is source-run / source-install
- [ ] release messaging says provider support is `Kimi --wire` only
- [ ] release messaging says local support is Windows and remote support is Linux
- [ ] release messaging does not claim installer, signing, auto-update,
      `Codex`, `Claude Code`, `attach`, or `P8` recovery

## 2. Core Docs Present

- [ ] `README.md` includes the minimal quick-start path
- [ ] `desktop/README.md` defines the desktop delivery baseline
- [ ] `remote-agent/README.md` defines the remote install surface
- [ ] `P6.5_trial_guide.md` defines the operator-led trial flow
- [ ] `P6.5_release_notes.md` exists and matches current scope
- [ ] `P6.5_launch_checklist.md` exists and matches current scope

## 3. Trial Entry Paths

- [ ] `README.md` points trial users to the correct quick-start entry
- [ ] `README.md` points operator-led trials to `P6.5_trial_guide.md`
- [ ] release notes point to the same entry docs

## 4. Feedback Intake

- [ ] `.github/ISSUE_TEMPLATE/bug_report.md` exists
- [ ] `.github/ISSUE_TEMPLATE/trial_feedback.md` exists
- [ ] `.github/ISSUE_TEMPLATE/environment_setup.md` exists
- [ ] `README.md` explains that Beta feedback goes through repo issues
- [ ] `P6.5_trial_guide.md` explains which template to use after a trial

## 5. Runtime Truthfulness

- [ ] docs still say local `relay` is started manually
- [ ] docs still say `desktop` is source-run only
- [ ] docs still say remote `remote-agent` needs manual env edits
- [ ] docs still say network checks remain manual
- [ ] docs still say `attach` is not implemented
- [ ] docs still say recovery / replay are not current Beta promises

## 6. Minimal Trial Path Sanity

- [ ] the documented commands still match the current repo layout
- [ ] the local relay command in docs still uses the current repo path and port
- [ ] the desktop command in docs still matches `desktop/` source-run
- [ ] the remote install command still matches
      `bash scripts/install-systemd-user.sh --start`
- [ ] the trial guide still includes one approval-prone task
- [ ] the trial guide still documents what to do if approval does not appear

## 7. Known Limitations

- [ ] release notes list known limitations explicitly
- [ ] README limitations still match release notes
- [ ] nothing in the docs upgrades contract-only behavior into a shipped promise

## 8. Launch Gate

- [ ] the outward-facing wording can be copied without correction
- [ ] a small external group can find the quick-start path
- [ ] a small external group can find the feedback path
- [ ] there are no remaining `P6.5` doc blockers for limited public Beta trial

If every item above is true, `P6.5` can be treated as content-complete for the
first public Beta and the next stage can move to `P7 Codex Support` without
reopening the Beta release surface.
